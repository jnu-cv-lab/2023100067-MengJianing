## 三、实验原理

### 1. 图像读取

在 OpenCV 中，可以使用 `imread()` 函数读取图像文件。
读取成功后，图像数据会保存在 `Mat` 类型变量中。

例如：

```cpp id="tbqzib"
Mat img = imread("text.png", IMREAD_COLOR);
```

其中：

* `text.png` 是读取的图像文件名
* `IMREAD_COLOR` 表示按彩色图方式读取

---

### 2. 图像基本信息获取

图像的宽度、高度、通道数等信息可以通过 `Mat` 对象获取：

* `img.cols`：图像宽度
* `img.rows`：图像高度
* `img.channels()`：图像通道数
* `img.type()`：图像数据类型

---

### 3. 图像显示

OpenCV 中可以通过 `imshow()` 显示图像，通过 `waitKey(0)` 等待按键关闭窗口。

---

### 4. 灰度图转换

彩色图像一般采用 BGR 三通道表示，而灰度图像只有一个通道。
可以使用 `cvtColor()` 将彩色图像转换为灰度图：

```cpp id="jpcqgo"
cvtColor(img, gray_img, COLOR_BGR2GRAY);
```

---

### 5. 图像保存

使用 `imwrite()` 可以将处理后的图像保存到本地文件中。

---

### 6. 像素访问与图像裁剪

* 可以通过 `img.at<>()` 访问某个位置的像素值
* 可以通过 `Rect()` 指定感兴趣区域（ROI），实现图像裁剪

## 六、结果分析

通过本次实验可以看出：

1. OpenCV 的 `Mat` 类型可以方便地存储和操作图像数据。
2. 彩色图像转换为灰度图后，图像只保留亮度信息，适合后续图像处理任务。
3. 使用 OpenCV 可以方便地读取图像属性、访问像素值以及裁剪图像区域。
4. 图像处理的基本流程通常包括：读取、显示、转换、保存和局部操作。

## 七、多文件编译说明

如果程序只有一个源文件，例如 `main.cpp`，可以直接编译：

### Linux / Mac（终端）

```bash id="u0vwwm"
g++ main.cpp -o main `pkg-config --cflags --libs opencv4`
```

运行：

```bash id="a8xwcl"
./main
```

---

如果程序包含多个源文件，例如：

* `main.cpp`
* `image_utils.cpp`
* `image_utils.h`

则可以这样编译：

```bash id="4djlwm"
g++ main.cpp image_utils.cpp -o main `pkg-config --cflags --libs opencv4`
```

运行：

```bash id="jlh0o9"
./main
```

---

在 Visual Studio 中，一般只需要：

1. 新建 C++ 项目
2. 配置 OpenCV 环境变量和库路径
3. 把多个 `.cpp` 文件一起加入工程
4. 直接点击运行即可
